Plugin Overview

Users Menu Manager Pro is a comprehensive WordPress plugin that gives you complete control over user roles, capabilities, and admin menu access. Perfect for multi-user websites, client portals, and businesses that need granular user permissions.

Form Access

Control access to Ninja Forms submissions and other form plugins.

Installation Guide

Follow these simple steps to install and activate Users Menu Manager Pro on your WordPress site.

1
Upload Plugin

Upload the plugin files to your WordPress plugins directory or use the WordPress admin to install.

2
Activate Plugin

Go to Plugins > Installed Plugins and click "Activate" for Users Menu Manager Pro.

3
Access Dashboard

Navigate to "Menu Manager Pro" in your WordPress admin menu to start configuring.

System Requirements
  • WordPress 5.0 or higher
  • PHP 7.4 or higher
  • MySQL 5.6 or higher
  • Administrator privileges for initial setup

Important Tips About Capabilities
  • Dependent Capabilities: Some capabilities automatically include others (e.g., 'edit_posts' includes 'read')
  • Plugin Dependencies: Plugin capabilities only work when the plugin is active
  • Security First: Start with minimal capabilities and add more as needed
  • Test Thoroughly: Always test user permissions before going live
  • Regular Review: Periodically review and update user capabilities

How to Use

Learn how to effectively use all the features of Users Menu Manager Pro.

Dashboard Overview

The main dashboard provides quick access to all plugin features:

  • Quick Stats: View total roles, users, forms viewers, and activity logs
  • Quick Actions: Direct links to manage roles, menus, forms access, and logs
  • User Management: Monitor administrators, editors, and recent user registrations
  • Recent Activity: Track all plugin activities and user actions

Roles Manager

Create and manage custom user roles:

  • Create New Role: Define role name, display name, and clone from existing roles
  • Capabilities: Select specific permissions from organized capability groups
  • Existing Roles: View, edit, and manage all current roles
  • Fix Existing Roles: Automatically add missing dependent capabilities

Menu Manager

Control admin menu visibility for different roles:

  • Role Selection: Choose which role to configure menu access for
  • Drag & Drop: Move menu items between available and hidden columns
  • Menu Preview: See exactly what menus will be visible for the selected role
  • Save Configuration: Apply menu restrictions and reset to defaults when needed

Ninja Forms Access

Manage access to Ninja Forms submissions:

  • Forms Viewer Management: Control who can view form submissions
  • User Assignment: Assign and remove users from forms viewer role
  • Access Control: Restrict access to only form submissions, not form creation
  • Role Integration: Seamlessly works with your custom roles and capabilities

Troubleshooting Guide

Common issues and their solutions to help you resolve problems quickly.

Problem: Plugin shows "Class not found" error or fails to load.
Solution:
  • Check if all plugin files are properly uploaded
  • Verify PHP version compatibility (7.4+)
  • Check WordPress error logs for specific error messages
  • Try deactivating and reactivating the plugin

Problem: Users see "Sorry, you are not allowed to access this page."
Solution:
  • Ensure user has the required capabilities
  • Check role assignments in WordPress Users section
  • Verify menu restrictions are not too restrictive
  • Check if user role has necessary permissions

Problem: Menu preview not showing or menus appearing in wrong location.
Solution:
  • Clear browser cache and refresh page
  • Check if JavaScript is enabled
  • Verify jQuery and jQuery UI are loaded
  • Check browser console for JavaScript errors

Problem: Users not appearing in Ninja Forms module or access issues.
Solution:
  • Ensure Ninja Forms plugin is active
  • Check if user has 'view_nf_submissions' capability
  • Verify user meta 'ummp_ninja_forms_access' is set
  • Try reassigning the forms viewer role

Problem: Database errors or tables not created.
Solution:
  • Check WordPress database connection
  • Verify database user permissions
  • Try deactivating and reactivating plugin
  • Check if database tables exist in wp_options

Quick Fix Tips

Clear Cache

Ensure your user account has administrator privileges.

Plugin Conflicts
